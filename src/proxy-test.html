7 · Capacitor / Mobile mobile

What is tested here: Everything a Capacitor WebView sends that differs from a desktop browser. Range requests (audio seeking), binary download integrity, origin-free requests, and large-response streaming. The on-device checklist below must be verified manually after the proxy goes live.
Manual on-device checklist — verify these after deploying the proxy and pointing the app at db.tafsirkurd.com:
  • iOS — Login (email + password)
  • iOS — Logout + session cleared from Keychain/localStorage
  • iOS — Session auto-refresh after 1 hour idle
  • iOS — Profile photo / image loading from Storage
  • iOS — Audio playback (Quran recitation) with seeking
  • iOS — PDF loading / mushaf page rendering
  • iOS — Push notification token registration
  • Android — Login (email + password)
  • Android — Logout
  • Android — Session auto-refresh
  • Android — Push notification token registration
  • Cloud sync — bookmarks / progress round-trip

8 · Supabase JS SDK Compatibility sdk

What is tested here: SDK-level behaviour that the proxy could silently break — token refresh flow, Set-Cookie passthrough, signed URL generation, onAuthStateChange event firing, localStorage key format, and PostgREST query builder correctness.

9 · Streaming / Large Files streaming

What is tested here: The Worker passes upstream.body (a ReadableStream) directly to the response — it never calls .arrayBuffer() or .json() on the body. These tests verify Range header support, 206 Partial Content responses, Accept-Ranges header presence, and time-to-first-byte vs. original.
