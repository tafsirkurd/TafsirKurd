2. SAVE TRANSLATION FUNCTION (Line 6474-6533)

   Function: async function saveTranslation(index)

   What it does:
   - Saves individual translation changes to Supabase database
   - Updates existing translations or inserts new ones
   - Provides visual feedback during save process
   - Handles errors gracefully

===============================================================================
HOW IT WORKS
===============================================================================

STEP 1: User clicks "💾 Save" button on a translation item

STEP 2: Button shows loading state
   - Text changes to: "⏳ Saving..."
   - Button is disabled

STEP 3: Function collects data from input fields:
   - Key ID: document.getElementById(`trans_key_${index}`)
   - Kurdish Text: document.getElementById(`trans_text_${index}`)
   - Context: document.getElementById(`trans_context_${index}`)
   - Page: document.getElementById(`trans_page_${index}`)
   - Active Status: document.getElementById(`trans_active_${index}`)

STEP 4: Database operation
   IF translation has an ID:
       - UPDATE existing record in Supabase
   ELSE:
       - INSERT new record in Supabase
       - Store returned ID

STEP 5: Success feedback
   - Button text: "✅ Saved!"
   - Background: Green gradient
   - Duration: 2 seconds
   - Then returns to normal state

STEP 6: Error handling (if any error occurs)
   - Button text: "❌ Error"
   - Background: Red gradient
   - Shows alert with error message
   - Duration: 2 seconds
   - Then returns to normal state

===============================================================================

   Color: Green gradient

   Text: "✅ Saved!"
   Color: Darker green (#4CAF50 to #45a049)
   Status: Disabled (for 2 seconds)

ERROR STATE:
   Text: "❌ Error"
   Color: Red gradient (#eb3349 to #f45c43)
   Status: Disabled (for 2 seconds)

===============================================================================
DATABASE OPERATIONS
===============================================================================

Table: kurdish_translations

UPDATE Operation (for existing translations):
   await supabase
       .from('kurdish_translations')
       .update(updatedTranslation)
       .eq('id', updatedTranslation.id);

INSERT Operation (for new translations):
   await supabase
       .from('kurdish_translations')
       .insert([updatedTranslation])
       .select();

===============================================================================
USE CASES
===============================================================================

1. Quick Edit: User can edit one translation and save it immediately
2. Testing: User can test changes on individual items before saving all
3. Safety: Reduces risk of losing all changes if something goes wrong
4. Performance: Faster than saving all translations when editing just one
5. Workflow: Better for making multiple small edits over time

===============================================================================
TECHNICAL DETAILS
===============================================================================

Function Signature:
   async function saveTranslation(index)

Parameters:
   - index: The index of the translation in translationsData array

Dependencies:
   - Supabase client (must be initialized)
   - translationsData array (global variable)
   - DOM elements with IDs: trans_key_${index}, trans_text_${index}, etc.

Error Handling:
   - try/catch block
   - Console logging for debugging
   - User alert for errors
   - Visual feedback on button

Return Value:
   - None (async function, updates UI directly)
